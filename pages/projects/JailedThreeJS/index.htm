Three.JS Made Easier!

JailedThreeJS is a DOM and CSS driven runtime on top of Three.js.

A <cell> becomes a 3D world where HTML tags create real THREE.Object3D instances, and CSS custom properties drive their state.

Here Is The Source For Wizz Dude:

HTML describes the scene graph:

CSS variables control almost any Object3D attribute

#wizzzard { --geometry: @cube; --position: (0,0,-1.5); --material-color: (0.8,0.2,0.2); --scale : (1,1,1); --transition: 400ms ease-in-out; }

Raycasting enables mesh interaction and pseudo states:

#wizzzard:hover { --scale : (1.2,1.2,1.2); --animation : spin360 0.5s linear 1; }

Even KeyFrames Work:

@keyframes spin360 { from { --rotation-y: 0; } to { --rotation-y: 6.283185; } }

I wanted 3D to feel web native.
Three.js is powerful, but most projects turn into JS boilerplate for setup, state, input, and animation.
I wanted a system where structure lives in HTML, animation and styling live in CSS, and you only write JS when you actually need logic.

In short this is not overly complicated and this is a more of a TL;DR Technical Overview :

->Bootstraps automatically: finds all <cell> elements and overlays a WebGL canvas


->Tag to constructor mapping: builds a map from THREE exports, so most Object3D subclasses are creatable as tags


->DOM to scene graph: scans children and adds created objects to the focused cell scene


->CSS to property paths: --a-b-c maps to object.a.b.c, with smart assignment for vectors and setters


->Pseudo states: raycast hits set flags like :hover, :active, :focus, then matching CSS rules repaint


->Transitions: --transition lerps numeric and array values with easing


->Keyframes: --animation plays CSS @keyframes by lerping custom properties over time


->Assets: CSS @AssetName rules define URLs, then @name resolves to loaded models, textures, audio, or materials


->Cross object referencing: values like #otherId-position-x can read from another object inside the same cell


->Reactivity: MutationObservers watch DOM and style changes and repaint the affected objects
