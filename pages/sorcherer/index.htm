WHAT is Sorcherer?

Sorcherer is a small Three.js helper library that lets you “pin” real HTML/DOM UI elements onto Object3Ds in your scene (labels, tooltips, images, HUD-ish bits), and keeps them synced as the camera/object moves.

WHY is Sorcherer?

Easier way to integrate DOM-elements into Three.js. It can allow for:

  • HTML (fonts, layout, inputs, accessibility, CSS) for UI and similar.
  • If you want those elements to behave like it’s “in” the 3D world (stick to objects, scale with distance, hide when off-screen/behind).
HOW does Sorcherer work?

Conceptually, it’s a per-frame sync step:

Register a mapping: Object3D DOM element (commonly via object.name / an id-matching scheme).

Each render tick:

  1. Compute each object’s world position.
  2. Project it through the camera to screen space (NDC → pixel coords).
  3. Update the overlay element’s CSS transform (translate to x/y).
  4. Optionally apply distance-based scaling so it shrinks/grows with depth.
  5. Optionally apply “simulate3D” transforms (limited; depends on implementation).
  6. Cull overlays that are out of view (and often ones “behind” the camera), to avoid DOM churn.

That’s it: a tight loop that turns 3D transforms into DOM transforms, plus culling + scale heuristics.
